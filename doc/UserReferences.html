VirtualGUser-1.3.5, what is it? home

VirtualGUser-1.3.5 is graphical user input/output emulation tool. It emulate mouse and keyboard input, let you change position and size of window, run and kill process, get the text from text- controls and much more.

It can be used as testing and operation system GUI automation tool for desktop operating systems. VirtualGUser is implemented as Lua- script interpreter (version 5.02) with some additional libraries and functionality.

The Microsoft Windows and the UNIX like operation systems with X11 based desktop with "Extended Window Manager Hints" support are implemented yet (gnome, kde, ... etc.). The script interface is cross platform, 1 time written – on all supported platforms can be executed.

The application work intern with native GUI objects (Window) and process. It can enumerate or find these objects based on title, relationships or desktop coordinates. It is implemented, simple to use, support to find out desktop coordinates. You can work with desktop or only with some window. Every time it is possible to save screenshot of whole desktop or other window. You can also compare the screnshot of a window with saved bitmap. If tesseract OCR command line tool is avaliable, you can use build in functions to get text from image file or from screen short.
You can run, test and kill all process with yours permissions. It is possible to find out the relationship between process and window.

The scripts depending only on software architecture of program that will be tested or software architecture of GUI on desktop of target operation system.

You do not need to use any foreign code in your software. You can test or automate foreign software.

It can be helpfully to show “End user” what is to do (like show me, support answer, tutorial creating) or in CI environment.

They are example scripts included. In the examples I show in simple form basics Ideas behind VirtualGUser.

If you need test or automation script, but you have no time, let us speak about.

The build of VirtualGUser is very simple. If you not found yours favorite operating system as binary download, try to do that.

Examples of use home

This examples are the files, which was used as test files during the development. I hope they will be usefully for you.

Usage: VirtualGUser-1.3.5 <LUA FILE> [-LUA FILE PARAMETER...]
    LUA FILE PARAMETER - is optional. It can be more than 1 if any.

calc.lua
It is the script to automate calculator on MSWindows and on UNIX like OS with GNOME desktop. It will try to find or run calc.exe or gcalctool. The script can handle command line arguments. It click the buttons and retrieve the result.
desk.lua
Cross platform script show desktop operation. It use coordinates recorded with recDesctop.lua.
editMain.lua
It is main file for default editor automation or testing. It call on MSWindows editWin.lua and on UNIX like OS editLin.lua.
editWin.lua
Search or run notepad.exe on MSWindows. Will be called from editMain.lua.
editLin.lua
Search or run gedit on UNIX like OS with GNOME desktop. Will be called from editMain.lua.
include.lua
In this file are some Lua function, which I use in other files.
libInfo.lua
This script enumerate all libraries and functions implemented by VirtualGUser.
rec.lua
The example record left mouse click position on calculator under MSWindows (calc.exe) and under UNIX like OS with GNOME desktop (gcalctool). The coordinates will be saved in recOut.txt.
recDesctop.lua
Same as rec.lua, but record the coordinates on desktop.
screen.lua
This script save screen shot and compare it whith saved.
